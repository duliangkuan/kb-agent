# 知识库 Agent · 测试方案（TEST_DESIGN）

> 本测试集用于验证"语义检索知识库 + 流式 RAG + MCP 工具"的真实能力。
> 方法论参考工业界标准 **BEIR**（检索测试集 qrels 标注格式）与 **RAGAS**（RAG 生成评测指标）。
> 设计原则：**每一篇素材、每一条查询都对应一个明确要验证的能力**，并刻意放入"硬负例"（hard negatives）制造区分度——离题文章太好区分等于没测。

---

## 1. 要验证的能力清单

| 编号 | 能力 | 含义 |
|---|---|---|
| A | 基础语义召回 | 搜主题词命中讲该主题的文 |
| B | 真语义 ≠ 关键词 | 搜文中**没出现**的同义说法也命中 |
| C | 多源召回 | 同主题多篇全部召回 |
| D | 跨文体召回 | 官方技术文 vs 观点随笔，文风差异大仍按主题召回 |
| E | 意图区分 | 同主题、要"事实"还是要"观点"命中不同文 |
| F | 抗干扰精准 | 近邻干扰/同词不同义/字面词不污染结果 |
| G | 多知识库隔离 | 限定某库 vs 跨全部库 |
| H | 多源综合（RAG） | 把对立/官方/多源综合成一个不偏的回答 |
| I | 拒答不编造 | 库里没有就答"未找到"，不幻觉 |
| J | 错误处理 | query 空 / 库不存在 / 检索失败 / 超时（题目硬要求） |
| K | 工具被 Agent 调用 | MCP 接入后自然语言自动触发 |

## 3. 相关性标注（qrels，分级 0/1/2）

借鉴 BEIR：每条查询标注哪些文档相关、相关到什么程度。**0=不相关，1=部分相关，2=高度相关**。未标注默认 0。分级相关性是 nDCG 有意义的前提。

## 6. 指标定义速查

| 指标 | 一句话 |
|---|---|
| **Recall@k** | 该找到的相关文，前 k 个里捞回来几成（漏没漏） |
| **Precision@k** | 前 k 个结果里相关的占比（准不准） |
| **MRR** | 第一个相关结果排在第几位的倒数（正确的靠前没） |
| **nDCG@k** | 相关度 × 位置权重的排序质量分，业界主指标 |
| **Faithfulness** | 答案只依据检索内容、不编造（防幻觉） |
| **Answer Relevancy** | 答案切不切题 |

## 7. 通过标准

- 检索类（T1-T6）：相关文档（rel≥2）应进 top_k；硬负例不得挤占 rel=2 文档的名次。
- 生成类（T1/T5/T7）：多源综合不偏向、拒答不编造。
- 错误处理（T8）：四类输入均返回可读错误、进程不崩。
- Agent（T9）：自然语言可触发工具并拿到可用结果。
- 多模态（M1/M2）：PDF 结果与 txt 一致、图噪不污染。
