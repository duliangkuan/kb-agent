# kb-grounded-writer · 有据写作 Skill

把「知识库 Agent」的检索能力，从"问答"延伸到"写作"：用本地知识库里的**真实语料**当唯一信源，模仿头部 AI 公众号 IP 的风格写稿；**找不到料就不写**，从机制上杜绝瞎编。

## ⚙️ 依赖（核心，必读）

本 Skill **不自带知识库，完全建立在本项目的 MCP server 之上**。运行前必须先把 `kb-agent` MCP 接入当前 Agent：

```json
{ "mcpServers": { "kb-agent": { "command": "python", "args": ["<绝对路径>/mcp_server.py"] } } }
```

本 Skill 会调用该 MCP 的两个工具：
- `search_knowledge_base(query, kb_id?, top_k?)` —— 语义检索语料（**写作素材的唯一来源**）
- `list_knowledge_bases()` —— 当用户没指定库时，先列出可用知识库

> 若 MCP 未接入或工具不可用：直接告知用户「未检测到 kb-agent 知识库工具，请先按 README 接入 MCP server」，不要凭记忆硬写。

## 触发 / 不触发

- **触发**：「用 X 的风格写一篇关于 Y 的（公众号/文章）」「基于知识库写 Y」「拿库里的料写篇 Y」「写一篇 Y，素材从知识库找」。
- **不触发**：脱离知识库的自由创作（那是普通写作）、只想"查"不想"写"（直接用 MCP 检索）、给定外部素材直接写（不需要本 Skill 的检索环节）。

## 工作流

### Step 1 · 解析请求
从用户输入中拆出两件事：
- **主题 Y**（要写什么）
- **目标风格**（卡兹克 / 宝玉 / 花叔 / 赛博禅心 / 风云……；未指定则问一句或用中性专业风）

### Step 2 · 检索语料（必经，决定能不能写）
- **默认检索全部知识库**：调用 `search_knowledge_base(query=主题 Y, top_k=5)` 时**省略 `kb_id`** 即为全库检索。
- **只有当用户明确说"只在《某库》里找 / 限定某个知识库"时**，才先 `list_knowledge_bases()` 拿到该库 `kb_id`，再带上 `kb_id` 调用。
- 主题较宽时，可拆成 2–3 个子查询分别检索，合并去重命中片段。

### Step 3 · 判定（关键护栏）
- **检索到相关片段**（有命中、相关度达标）→ 进入 Step 4 写作。
- **检索为空 / 全部为弱相关** → **不写**，按下方「拒写规范」回复。
- 判断标准：以 MCP 返回结果为准；返回"未找到相关内容"或只剩明显离题片段，即视为无料。

### Step 4 · 有据写作
- **只用检索到的真实素材**（事实、观点、数据、案例）作为内容来源，**不得引入知识库之外的虚构事实**；常识性串联可以，编造事实/数据不行。
- 按目标风格组织成一篇完整公众号文（标题 + 开头钩子 + 正文 + 收尾）。
- **标注来源**：文末或行内注明引用了知识库里的哪几篇（用 search 返回的标题）。
- 若多篇语料观点对立（如「微信小程序 AI 化」的支持/反对），如实呈现多方、不偏向——这正是知识库多源检索的价值。

## 风格库（voice 速查，可对接已有 writer skill）

| IP | voice 关键词 |
|---|---|
| 数字生命卡兹克 | 实测体验派、场景化叙事、技术+娱乐化、口语化金句 |
| 宝玉 | 冷静技术解读、翻译+点评、第一性原理、克制不煽情 |
| 花叔 | 一人公司视角、产品实战、emotion 充沛、敢从 0 到 1 |
| 赛博禅心 | AI 科普 + 工具实测、轻松有梗、产品横评 |

> 若本机装有对应 writer skill（如 `khazix-writer` / `fengyun-writer` / 花叔系 skill），优先委托其执行 Step 4 的文风落地，本 Skill 只负责"检索取料 + 拒写护栏 + 把素材喂给它"。

## 拒写规范（找不到料时）

不要硬写。按此模板回复（替换 X 与库名）：

> 知识库中没有检索到关于「**X**」的相关素材，所以这篇我写不了——本 Skill 只基于你本地知识库里的真实资料写作，不编造。
> 解决办法：把关于「X」的资料（公众号文章、PDF、笔记等）上传到知识库后再让我写。可在网页端 `http://localhost:8000/` 上传，或让我调用 `add_text_to_kb` 直接入库。

## 错误处理

- **MCP 未接入 / 工具不存在** → 提示先接入 kb-agent MCP，不硬写。
- **检索失败 / 超时**（MCP 返回错误文本）→ 原样转达错误并建议重试，不把报错当成"无料"而误触发拒写文案。
- **query 为空 / 主题不明** → 先问清楚要写什么主题、什么风格。

## 示例

**例 1 · 有料**
用户：「用卡兹克的风格写一篇微信小程序 AI 化的稿子」
→ `search_knowledge_base("微信小程序 AI 化", top_k=5)` 命中官方/卡兹克/宝玉三篇 → 以卡兹克实测体验风成文，呈现支持与反对两方观点 → 文末标注引用来源。

**例 2 · 无料**
用户：「写一篇关于量子计算最新突破的公众号」
→ 检索知识库无相关命中 → 触发拒写文案，建议先上传量子计算相关资料。
