# 实现思路说明

## 一、整体设计

系统分两层，**共享同一个检索内核**：

- **第一部分（知识库）**：FastAPI 提供知识库 CRUD、文档上传、语义检索、SSE 流式 RAG。
- **第二部分（Agent）**：`mcp_server.py` 把检索能力封装成 MCP 工具，供外部 Agent 调用。

两层都调用 `app/retriever.py` 这一份检索逻辑——**单一数据源、单一真相**，不写两套，避免实现漂移。这也是本项目最重要的工程取舍。

---

## 二、技术选型理由

| 选型 | 理由 |
|---|---|
| **FastAPI** | 原生异步 + 自带 SSE（StreamingResponse），适合流式返回 |
| **SQLite + numpy** | embedding 以 JSON 存进 SQLite，检索时用 numpy 算 cosine。**刻意不用 sqlite-vec**——免去 Windows 装扩展的坑，且本题规模下全表 cosine 足够快，部署零依赖 |
| **bge-m3 + bge-reranker-v2-m3** | 中文语义检索强；两阶段（召回 + 精排）比单向量更准。bge-m3 是 instruction-free，查询**不加前缀** |
| **DeepSeek-V3** | 流式生成质量高，且与硅基流动同一个 key，省配置 |
| **FastMCP** | 官方 MCP SDK，stdio 传输，能被 Claude Desktop/Cursor/Claude Code 直接接入，正对 Agent 工程师题意 |
| **TextIn** | 通用文档解析（版面分析+表格+OCR），让上传支持 PDF/Word/图片，是“多模态”加分路径；txt/文本为保底 |

---

## 三、检索流程（retriever.py）

**入库**：文本 → 分块（默认 400 字、重叠 80）→ bge-m3 向量化 → 连同元数据写入 `chunks` 表。

**检索（两阶段 + 两道过滤）**：
1. **向量召回**：query 向量化，对全库 chunk 算 cosine，取 Top-N（默认 10），并用 `SIM_THRESHOLD` 过滤明显无关者（召回阶段）。
2. **reranker 精排**：用 bge-reranker 对候选重排，得到更准的相关度。reranker 调用失败时**降级**为向量结果，不让检索崩。
3. **文档去重**：同一篇文章可能多个分块命中，只保留最相关的一块——避免长文用多个分块霸占名额，让结果是 *top_k 篇不同文章*。
4. **精排相关度下限**（`RERANK_MIN_SCORE`）：低于阈值视为不相关，既不展示也不作答——这保证了“拒答时不显来源”的一致性（RAG 的 no-answer 阈值思想）。

> 第 3、4 步是联调中通过自动测试发现并修复的：去重解决了“长文洪水”，下限解决了“噪声块混入命中来源”。

---

## 四、流式返回（SSE）

`GET /api/search/stream` 用 SSE 分事件推送：
- `sources` 事件：先把命中来源（标题 + 相关度 + 原文片段）一次性发给前端；
- `token` 事件：再把 DeepSeek-V3 基于命中内容生成的回答**逐字**推送；
- `done` / `error` 事件：收尾或报错。

响应头带 `X-Accel-Buffering: no`，每条事件以 `\n\n` 结尾，保证“逐字蹦出”不被缓冲。

---

## 五、Agent 工具化（MCP）

`mcp_server.py` 用 FastMCP 暴露三个工具：
- `search_knowledge_base(query, kb_id?, top_k?)` — 语义检索，返回命中内容 + 来源
- `list_knowledge_bases()` — 列出知识库供 Agent 发现
- `add_text_to_kb(kb_id, title, content)` — 向库追加文本

**错误处理**（题目硬要求）：query 为空 / 知识库不存在 / 检索失败 / 调用超时，全部返回**可读错误文本**而非抛异常——保证 stdio 进程不被打断；并用 `asyncio.wait_for` 设超时上限。

---

## 六、拒答机制（可信度）

生成 prompt 明确要求“只依据参考资料回答，没有就答『知识库中未找到相关内容』，绝不编造”；叠加检索的相似度阈值 + 精排下限，库里没有的问题会**如实拒答**，不产生幻觉。

---

## 七、测试与已知边界

设计了一套参考 BEIR（qrels 分级标注）+ RAGAS（生成指标）的测试集（见 `TEST_DESIGN.md`），并提供自动评测脚本 `scripts/run_tests.py`。核心结论：语义检索、多源综合（能同时召回并综合对立观点）、意图区分、抗干扰、拒答、错误处理均通过。

**诚实记录的边界**：极端跨域改写（如用“轻量级应用”指代“小程序”）时 bge-m3 桥接较弱，此时系统因低于精排下限而**诚实拒答**，而非返回弱匹配——这是可接受的精确度取舍，也是后续可优化方向（如查询改写、混合检索）。
