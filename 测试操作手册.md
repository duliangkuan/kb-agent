# 知识库 Agent · 测试操作手册（手动执行版）

> 配合 `TEST_DESIGN.md`（测试设计依据）使用。本手册是**一步步照着点就能跑完**的操作指南，每条测试给出：在哪操作、输入什么、看什么、预期结果、☐通过/☐不通过 记录栏。
> 测试分两阶段：**阶段一·基础**（上传 txt，不依赖 TextIn）→ **阶段二·多模态**（上传 PDF，验证 TextIn 解析）。

---

## 〇、启动准备

1. 配置 `.env`：确保 `SILICONFLOW_API_KEY` 已填（TextIn 凭证用于阶段二）。
2. 命令行进入项目目录，执行：
   ```
   pip install -r requirements.txt
   uvicorn app.main:app
   ```
3. 浏览器打开 **http://localhost:8000/** ，看到交互页即就绪。
4. （可选）每次想从干净状态开始，可停服务、删除 `data/kb.db` 后重启。

## 二、阶段一 · 检索测试（逐条照做）

> 每条：在右侧检索框输入【查询】，设好【范围】，回车。看【命中来源】（标题+相关度+原文片段）和【回答】。

### ✅ T1　综合检索 + 多源综合
- **范围**：勾选「跨全部知识库检索」
- **查询**：`微信小程序到底该不该AI化？官方什么态度，业内有哪些不同看法？`
- **看什么**：命中来源里是否出现 **A1 官方、A2 卡兹克、A3 宝玉** 三篇；回答是否分别讲了三方立场
- **预期**：三篇都命中；回答综合「官方两种模式 / 卡兹克支持 / 宝玉反对」，不偏向
- **记录**：☐通过　☐不通过　实际：________________

### ✅ T2　换词语义（最关键，证明不是关键词匹配）
- **范围**：跨全部库
- **查询**：`轻量级应用要不要接入大模型能力？把对话式AI塞进去值不值？`（**故意不出现"微信小程序AI化"字样**）
- **看什么**：是否仍命中 A2、A3（及 A1）
- **预期**：仍命中那几篇 → 证明是语义理解，不是字面匹配
- **记录**：☐通过　☐不通过　实际：________________

### ✅ T3　意图区分（要"事实"命中官方文）
- **范围**：限定 KB-A（点选 KB-A，不勾跨库）
- **查询**：`开发者具体怎么接入微信的AI能力？有哪些模式？`
- **看什么**：命中来源**第一条**是否是 A1 官方指引
- **预期**：A1 排第一（要 how-to 优先命中事实文），观点文在后
- **记录**：☐通过　☐不通过　实际：________________

### ✅ T4　抗干扰精准 / 反向证明

- **看什么**：命中来源是否是 B2（额度技巧）；**小程序三篇 A1/A2/A3 不应出现在前列**
- **预期**：命中 B2，小程序文不串进来 → 证明精准、不乱吐
- **记录**：☐通过　☐不通过　实际：________________

### ✅ T5　第二主题多源召回

- **看什么**：是否命中 B4（Cloudflare裁员）+ B5（反AI浪潮）；回答是否综合两篇
- **预期**：两篇都召回并综合
